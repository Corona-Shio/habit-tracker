# Notion用レシピMarkdown

## 概要

雑多なレシピメモを、Notionで読みやすいMarkdown構造に整える。ユーザーが指定した見出し順とラベルは保ち、材料と栄養成分は表に統一する。

## ワークフロー

### 1) 入力を読み取りセクションに分解

レシピ名と、ユーザーが用意した各セクション（例: 材料、作り方、栄養成分、備考、コツ、代用、保存方法）を特定する。順番はユーザーの提示どおりにする。

### 2) Markdownの骨組みを作る

使用ルール:
- タイトルはH2（`##`）。ユーザーが別の見出しレベルを指定している場合はそれに従う。
- セクション見出しはH3（`###`）。ユーザー指定がある場合はそれに従う。
- 材料と栄養成分は必ず表にする。
- 作り方は番号付きリストにする。

### 3) 内容を整える

- 絵文字、太字、ラベルは可能な限り維持する。
- 分量と単位は表記を変えない（括弧内も含む）。
- 不足情報を捏造しない。材料や作り方が欠ける場合は短く確認する。
- 複数レシピがある場合はそれぞれ独立して整形し、`---` で区切る。

## 出力ルール

- **タイトル**: ユーザーのレシピ名をそのまま使う。名前がない場合は確認する。
- **見出し表記**: 出力する見出しは、先頭に絵文字を付け、末尾を角括弧で閉じる（例: `## 🍽️【アボカドソース】`）。
- **見出しゆれ対応**: 意味が同じ見出し（例: 備考/その他/メモ）は同じ絵文字を使う。対応表にない見出しは意味の近い既存カテゴリへ寄せる。
- **材料**: 必ず2列の表にする。列名はユーザー指定があればそれを使う（例: 材料/分量、食材/分量）。
- **栄養成分**: 必ず2列の表にする（列名は基本「栄養素/数値」、ユーザー指定があればそれに従う）。
- **作り方**: 番号付きリスト。ユーザーの強調（太字など）は保持する。
- **その他セクション**: H3見出し + 文章/リストで、ユーザーの内容に合わせて整形する。

## 見出し絵文字対応表

- 🍽️: タイトル
- 🧂: 材料
- 🥣: 作り方
- 🧪: 栄養成分
- 💡: コツ
- 📝: 備考 / その他 / メモ / 注意点
- 🔁: 代用
- 🧊: 保存方法

## クイック例

入力（自由形式）:
- タイトル: アボカドソース
- 材料: アボカド 1個、塩 ひとつまみ
- 作り方: 1) 混ぜる 2) 完成

出力（Markdown）:
```md
## 🍽️【アボカドソース】

### 🧂【材料】

| 材料 | 分量 |
| --- | --- |
| アボカド | 1個 |
| 塩 | ひとつまみ |

---

### 🥣【作り方】

1. 混ぜる
2. 完成
```
